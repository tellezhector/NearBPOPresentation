Complejidad computacional


Una manera de medir la eficiencia de un algoritmo.

  • ¿Cuántas operaciones requiere el algoritmo con respecto al tamaño de la entrada?
  • ¿Qué es una operación?

Complejidad computacional


Una operación es
  • asignación
  • comparación
  • operación aritmética
  • operación booleana

Ejemplo:


int binary_search(int A[], int key)
{
  int imin = 0;
  int imax = A.length;
  while (imax >= imin)                     //una operación
  {
	int imid = midpoint(imin, imax);  //dos operaciones
	if(A[imid] == key)                //una operación
	  return imid;                    
	else if (A[imid] < key)           //una operación
	  imin = imid + 1;                //una operación
	else                 
	  imax = imid - 1;                 //una operación
  }
  return KEY_NOT_FOUND;                 
}

Complejidad computacional


Dependiendo de la entrada, un algoritmo puede necesitar más o menos operaciones para completarse.

Podemos definir una función que dependiendo del tamaño de la entrada nos diga el número de operaciones que necesitará determinado algoritmo.

Ejemplo:


int binary_search(int A[], int key)
{
  int imin = 0;
  int imax = A.length;
  while (imax >= imin)                
  {
	int imid = midpoint(imin, imax);  
	if(A[imid] == key)                
	  return imid;                    
	else if (A[imid] < key)           //Si no se encuentra key,
	  imin = imid + 1;                //la distancia entre imin e imax 
	else                              //disminuye por la mitad
	  imax = imid - 1;                 
  }
  return KEY_NOT_FOUND;                 
}
binarySearch es \(O(log(n))\) (léase orden de lógaritmo de \(n\)) donde \(n\) es el tamaño de la entrada (el tamaño del arreglo en esta caso).

En análisis asintótico, lo que importa es como cambia el radio o razón de crecimiento.

Por ejemplo, los radios de crecimiento de las funciones \(f(n) = n\) y \(g(n) = 2n\) son constantes, y por lo tanto se considera que son del mismo orden.

Pero los radios de crecimiento de las funciones \(f(n) = n\) y \(h(n) = 2^n\). son por un lado \(\frac{n}{2^n} \rightarrow 0\) y por otro lado \(\frac{2^n}{n} \rightarrow \infty\), y por lo tanto \(O(2^n)\) es un orden mucho más grande que \(O(n)\).

Complejidad computacional:


A los computólogs les gustan los algoritmos de orden polinomial. (En la industria, polinomial no siempre es suficiente).

Recuerden que un polinomio es algo de la forma

\(p(n) = a_0 + a_1n + a_2n^2 + a_3n^3 + \dots + a_mn^m\)

el grado de un polinomio es el el grado más alto de sus términos, en este caso, el grado es \(m\).

Ejemplo:

(acoplamientos estables)

Imagine que existe un conjunto de \(n\) hombres y otro de \(n\) mujeres. Cada hombre tiene una lista de preferencias de las mujeres y cada mujer una lista de preferencias de los hombres.

Ejemplo:

(acoplamientos estables)

Queremos asignar parejas sin que queden parejas "inestables", es decir, un hombre y una mujer que se prefieran más entre ellos que a las parejas que se les asignó.

Ejemplo:

Algoritmo Gale-Shapley 

function stableMatching {
    Initialize all m ∈ M and w ∈ W to free
    while ∃ free man m who still has a woman w to propose to {
       w = highest ranked woman to whom m has not yet proposed
       if w is free
         (m, w) become engaged
       else                   // some pair (m_2, w) already exists
         if w prefers m to m_2
           (m, w) become engaged
           m_2 becomes free
    }
}
Este algoritmo es en peor caso cuadrático, pues cada hombre podría tener que hacer todas sus \(n\) propuestas y esto para cada uno de los \(n\) hombres, este algoritmo es \(O(n^2)\).

Clases de problemas


Los problemas que se pueden solucionar en tiempo polinomial, se dice que pertenecen a la clase de problemas \(P\).

Existe otra clase de problemas a la que se le llama \(NP\), esta es la clase de los problemas que se pueden resolver en tiempo polinomial en una computadora que puede "clonarse a sí misma".

Ejemplo:

(de un problema en \(NP\))

En una ciudad con \(n\) personas, decir si existen \(k\) que no se conozcan entre sí. Donde (\(2 \leq k \leq n\)).

Este problema es conocido como "Independent Set Problem" y no sólo pertenece \(NP\), cualquier otro problema que también pertenezca a \(NP\) puede reducirce a él. A estos problemas se les conoce como \(NP\)-completos. Si pueden encontrar un algoritmo polinomial para solucionar cualquier problema \(NP\)-completo se ganan un millón de dólares y se vuelven muy famosos.

Programación Entera


De nuevo tenemos una ciudad con \(n\) habitantes pero esta vez queremos encontrar el grupo de personas más grande tal que no se conozcan entre sí.

Cada habitante puede ser representado en una dimensión distinta. Y cada conjunto de personas puede ser representado por un vector de unos y ceros dependiendo de si está o no la persona en el conjunto.

e.g. el véctor \((0, 1, 1, 0)\) representa al conjunto que tiene a las personas 2 y 3.

Programación Entera

Podríamos tomar la envolvente convexa de todos estos y entonces hacer programación lineal sobre él, pero no siempre es fácil describir estas envolventes, de hecho, en el caso general, la de este problema (el de la ciudad donde buscamos personas que no se conozcan) es desconocida.

Esta variante se llama programación entera porque esperamos tener respuestas con números enteros. No tiene sentido para nosotros si obtuvieramos una respuesta que nos diga "toma la mitad de la persona 5 y tres cuartas partes de la 9".

Propiedad de intersección

Gilberto Calvillo Vives

Con la idea de clasificar problemas, estudió propiedades de intersección de poliedros.

Su contribución (a la que nos enfocamos) dice más o menos: Si el poliedro que buscas tiene (algunas) rebanadas enteras, entonces el problema que representa no es (tan) difícil.

Mostró que el Independent Set Problem satisface estas condiciones bajo una restricción. Esto llevó a más tarde dar una solución polinomial para los escenarios que tuvieran esta restricción.

Ejemplo:


Pensemos en el problema de las parejas estables y, cada posible pareja la representamos como la entrada de un vector de ceros y unos (cero si no asignamos esa pareja y uno si sí).

Por un lado, tomemos aquellos en los que a cada hombre únicamente le toca una mujer. Estos generan un poliedro.

Por otro lado, tomemos aquellos en los que a cada mujer únicamente le toca un un hombre. Estos generan otro poliedro.

La intersección de estos dos poliedros es el poliedro de los acoplamientos posibles. Este resulta tener la propiedad de intersección de Gilberto Calvillo y por lo tanto resulta no ser tan difícil.

Esto no es nada nuevo, este problema es polinomial y es sabido desde hace mucho tiempo.
